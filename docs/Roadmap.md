# Roadmap

## Purpose
Organize JOHAI work into stabilization, intelligence, SaaS expansion, and channel delivery phases.

## Architecture
The roadmap grows in layers: stabilize the core, strengthen AI reasoning, complete multi-tenant ownership, then add delivery channels and automation depth.

## Components
Product stabilization, dashboard componentization, multi-tenant ownership, knowledge embeddings, AI automation actions, external channels, and billing.

## Current Status
Completed foundation includes CRM, dashboard authentication, lead cleanup, Calendly, email, follow-ups, multi-tenant schema preparation, onboarding, Knowledge Center, document ingestion architecture, semantic memory architecture, AI Orchestrator, Business Brain, Audit Engine, Morning Brief, Chief of Staff, and Command Center.

## Future Roadmap
1. Split dashboard into workspace components.
2. Add production Supabase Auth business users.
3. Implement embeddings and pgvector search.
4. Add scheduled Morning Brief delivery.
5. Add Slack, Teams, SMS, WhatsApp, and voice.
6. Add subscription billing.
7. Add tenant-specific AI assistant configuration.

## Dependencies
Supabase migrations, dashboard stability, provider environment variables, and external channel APIs.

## Known Limitations
Some roadmap items require database migrations, RLS review, production credentials, and webhook verification.
